# Current Session: Fix 5 Persistent UI/Data Issues

## Goal
Fix 3D rendering (unclear/poor), broken unit label visibility toggle, "building" text displayed next to unit codes, ProjectSitePlan Append/Replace buttons, and GeoJSON-to-unit sync failure.

## Progress

### Fixed & Deployed (commit `635a2a3`, pushed to `main`)
- **Label toggle rewritten** – `UnitClusterLayer` (MapsPage) and `UnitMarkers` (ProjectSitePlan) now use **two independent `useEffect`s**: one for marker cluster (`[map, units, onUnitClick]`), one for label layer (`[map, units, showLabels]`). The label layer is a standalone `L.layerGroup` added after the cluster so it's always on top. Labels re-render when `units` or `showLabels` changes.
- **Default imported type changed `'building'` → `'unit'`** – both `handleGeoJSONUpload` and `processGeoJson` use `'unit'` as fallback when no `properties.type` exists. "Feature"/"FeatureCollection" values are remapped to `'unit'`.
- **GeoJSON-to-units sync expanded** – condition changed from `geoType === 'unit'` to `geoType !== 'site'`; all non-site imported features sync directly to `units` table.
- **3D view improved** – floor height 0.35→1.0, box caps doubled (8→16, 6→12), edge opacity increased (site 0.25→0.5, floor 0.08→0.2, building 1→1), added backlight, directional light intensity 1.5→2.0, `ACESFilmicToneMapping` enabled, minimum box sizes raised.
- **Append/Replace buttons** – added `unitRefreshKey` state incremented on `generate_project_units` success, passed as `refreshKey` prop to `UnitMarkers` to trigger unit re-fetch.
- Deployed to `https://alfanar-erp.pages.dev` (MapsPage chunk `BjWCh4pc`, was `CzEFa8uF`).

### Not Yet Tested
All fixes are on production but **user has not confirmed** after hard-refresh (Ctrl+Shift+R). Previous round was also deployed but user reported "same issues." These are deeper fixes, but need verification.

## Key Decisions
- **Label toggle uses separate `useEffect` with `[map, units, showLabels]`** – previous approach had toggle effect without `units` dep, so units refresh lost labels.
- **Default type `'unit'`** – most GeoJSON imports are unit-level; 'building' default produced wrong labels and blocked sync.
- **Sync all non-site** – site boundaries are the only geometry that should NOT become a unit record.
- **3D box sizes doubled** – original caps were too small relative to camera distance.
