## 🚀 الخطوة 1: إنشاء حساب Supabase

> Supabase هو قاعدة بيانات تطبيقك + نظام تسجيل الدخول + رفع الملفات. كل شيء مجاني للبدء.

1. اذهب إلى موقع: **[https://supabase.com](https://supabase.com)**
2. اضغط على **"Start your project"** (الأزرق الكبير)
3. سجل حساب باستخدام **GitHub** (أسهل) أو Gmail
   - إذا اخترت GitHub: ستحتاج حساب GitHub (مجاني في [github.com](https://github.com))
4. بعد تسجيل الدخول، سيطلب منك إنشاء **منظمة (Organization)**
   - ادخل أي اسم مثل `شركتي`
   - اضغط **"Create organization"**
5. الآن اضغط **"New project"**
6. املأ البيانات:
   - **Name**: `erp-construction` (أو أي اسم تريده)
   - **Database Password**: اختر كلمة سر قوية واحفظها في مفكرة (مثلاً `MyStr0ng!Pass123`)
   - **Region**: اختر الأقرب لك (مثلاً `South Asia (Mumbai)` أو `EU West`)
   - اضغط **"Create new project"**
7. **انتظر 2-3 دقائق** حتى يتم إنشاء المشروع 🕐

![Supabase New Project](https://supabase.com/docs/img/supabase-new-project.png)

## 🗄️ الخطوة 2: إنشاء قاعدة البيانات

هذه الخطوة最关键 - سننشئ 73 جدولاً و 153 سياسة أمان بنقرة واحدة.

1. في صفحة مشروع Supabase، اذهب للقائمة اليسرى → **"SQL Editor"**
2. اضغط على **"New query"**
3. **افتح ملف قاعدة البيانات** الموجود في مجلد `database` الذي نزلته مع المشروع:
   - اذهب لملف `database/999_full_migration.sql`
   - افتحه في **المفكرة (Notepad)** 
   - حدد كل النص (Ctrl+A) وانسخه (Ctrl+C)
4. ارجع لـ Supabase والصق النص (Ctrl+V) في مربع SQL Editor
5. اضغط على **"Run"** ▶️ أو `Ctrl+Enter`
6. **انتظر 10-30 ثانية** حتى تظهر رسالة النجاح ✅
7. خذ راحة ☕ - لقد أنشأت 73 جدولاً!

8. الآن **كرر نفس الخطوات** مع ملف `supabase/migrations/seed_data.sql`:
   - افتحه، انسخ، الصق، Run

## 🔗 الخطوة 3: ربط التطبيق بقاعدة البيانات

### 3.1 تحميل المشروع على حاسوبك

> ستحتاج تحميل مجلد المشروع على حاسوبك أولاً.

**الطريقة الأسهل - تحميل كملف ZIP:**
1. اذهب إلى رابط المشروع (إذا أعطاك إياه المطور) أو استخدم الملفات الموجودة في مجلد `ERP`
2. الملفات موجودة بالفعل على حاسوبك في المسار: `D:\OpenCode\ERP\`

### 3.2 تثبيت Node.js

1. اذهب إلى [nodejs.org](https://nodejs.org)
2. حمل الإصدار **LTS** (الزر الأيسر الكبير)
3. افتح الملف الذي نزلته واتبع التعليمات (next → next → install)
4. **أعد تشغيل الحاسوب** بعد التثبيت

### 3.3 الحصول على مفاتيح Supabase

1. اذهب لصفحة مشروعك في Supabase
2. من القائمة اليسرى → **"Project Settings"** ⚙️ (آخر أيقونة)
3. اذهب لـ **"API"** في القائمة اليسرى
4. ستجد معلومتين مهمتين:
   - **Project URL** (ينتهي بـ `.supabase.co`)
   - **anon public key** (نص طويل)
5. **انسخهما في المفكرة**، سنحتاجهما بعد قليل

![Supabase API Settings](https://supabase.com/docs/_next/image?url=%2Fdocs%2Fimg%2Fsupabase-client-url.png&w=1920&q=75)

### 3.4 إعداد ملف البيئة

1. اذهب إلى مجلد `erp-frontend` على حاسوبك
2. ستجد ملف اسمه `.env.example`
3. **انسخ الملف** وأعد تسمية النسخة إلى `.env` (فقط `.env` بدون أي امتداد)
   - في المفكرة: افتح `.env.example` واحفظ باسم `.env`
4. افتح ملف `.env` في المفكرة
5. سيبدو هكذا:
   ```
   VITE_SUPABASE_URL=https://your-project.supabase.co
   VITE_SUPABASE_ANON_KEY=your-anon-key
   ```
6. **استبدل النصوص** بما نسخته من Supabase:
   ```
   VITE_SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   ```
7. **احفظ الملف** (Ctrl+S)

### 3.5 تشغيل التطبيق

1. **افتح PowerShell** (اضغط كليك يمين على زر Start → Windows PowerShell أو Terminal)
2. اكتب الأوامر التالية (اضغط Enter بعد كل سطر):

```powershell
cd D:\OpenCode\ERP\erp-frontend
npm install
npm run dev
```

3. **انتظر** حتى يظهر نص مثل:
   ```
   VITE v5.x.x  ready in xxx ms
   ➜  Local:   http://localhost:5173/
   ```
4. **اضغط** على الرابط `http://localhost:5173/` (اضغط Ctrl + كليك على الرابط)
5. **مبروك!** التطبيق يعمل على حاسوبك! 🎉

> ⚠️ **ملاحظة**: أول مرة ستفتحها، ستنتقل لصفحة تسجيل الدخول. هذا طبيعي.

## 🌐 الخطوة 4: رفع التطبيق على الإنترنت (Cloudflare Pages)

> الآن سننشر التطبيق ليكون متاحاً للجميع عبر الإنترنت مجاناً.

### 4.1 بناء التطبيق

1. في PowerShell، أوقف التطبيق بالضغط على `Ctrl+C`
2. اكتب:

```powershell
npm run build
```

3. انتظر حتى ينتهي البناء. سيظهر مجلد جديد باسم `dist`

### 4.2 إنشاء حساب Cloudflare

1. اذهب إلى [https://dash.cloudflare.com/sign-up](https://dash.cloudflare.com/sign-up)
2. سجل حساب جديد (مجاني)
3. أكد بريدك الإلكتروني

### 4.3 رفع التطبيق على Cloudflare Pages

1. من لوحة تحكم Cloudflare، اذهب للقائمة اليسرى → **"Pages"**
2. اضغط على **"Create a project"** → **"Upload directly"** (أو "Direct Upload")
3. **أدخل اسم المشروع**: `erp-construction`
4. **اسحب وأفلت** مجلد `dist` من حاسوبك إلى منطقة السحب
   - مجلد `dist` موجود في `D:\OpenCode\ERP\erp-frontend\dist`
5. انتظر 30-60 ثانية
6. **مبروك!** تطبيقك الآن على الرابط التالي:
   ```
   https://erp-construction.pages.dev
   ```
7. **شارك الرابط** مع فريقك!

> 💡 **نصيحة**: يمكنك ربط اسم نطاق خاص بك (مثل `erp.sharikati.com`) من إعدادات المشروع في Cloudflare Pages.

## 👑 الخطوة 5: إنشاء أول مستخدم مشرف

1. افتح تطبيقك (على `localhost:5173` أو رابط Cloudflare)
2. ستظهر صفحة تسجيل الدخول
3. في شاشة تسجيل الدخول، **ليس هناك زر تسجيل** - هذا مقصود للأمان
4. **لإنشاء أول مستخدم، اذهب إلى Supabase Dashboard:**
   - القائمة اليسرى → **"Authentication"** → **"Users"**
   - اضغط **"Add user"**
   - أدخل بريد إلكتروني وكلمة سر (مثال: `admin@sharikati.com` / `Admin123!`)
   - اضغط **"Create user"**
5. **الآن اربط المستخدم بصلاحية المشرف:**
   - اذهب للقائمة اليسرى → **"SQL Editor"**
   - افتح استعلام جديد واكتب:

```sql
-- استبدل USER-ID بالرقم الذي ظهر بعد إنشاء المستخدم
INSERT INTO user_profiles (id, email, full_name_en, full_name_ar, role)
VALUES ('USER-ID', 'admin@sharikati.com', 'System Admin', 'مدير النظام', 'admin');
```

6. الآن **اذهب لصفحة تسجيل الدخول** في تطبيقك
7. أدخل البريد وكلمة السر واضغط **"Sign In"**
8. **مبروك!** أنت الآن مشرف على النظام! 🎉

## 📧 الخطوة 6: تشغيل البريد الإلكتروني (اختياري)

### تفعيل تسجيل المستخدمين الجدد

1. في Supabase Dashboard → **Authentication** → **Settings**
2. تحت **"Email Auth"**، تأكد أن **"Enable email confirmations"** نشط
3. تحت **"SMTP Settings"**، فعّل **"Custom SMTP"**
4. أدخل إعدادات Gmail أو أي مزود بريد:
   - يمكنك استخدام **SendGrid** (مجاني 100 إيميل/يوم)
   - أو **Gmail SMTP** (يتطلب كلمة سر تطبيق خاص)
5. اضغط **"Save"**

## 🎯 الخطوة 7: بدء استخدام النظام

### بعد تسجيل الدخول كمشرف:

1. **اذهب لـ "Settings" ← "System Designer"** من القائمة الجانبية
2. هنا يمكنك:
   - **تشغيل/إيقاف** الوحدات (Modules)
   - **تعديل حالات** كل وحدة (Statuses)
   - **إنشاء سير عمل** (Workflows)
   - **إضافة حقول مخصصة** (Custom Fields)
   - **تصميم مؤشرات أداء** (KPIs)

### إضافة مشروعك الأول:

1. اذهب لـ **"Projects"** من القائمة الجانبية
2. اضغط **"Create"**
3. أدخل اسم المشروع وبياناته
4. بعد إنشاء المشروع، اذهب لـ **SQL Editor** وأضف نفسك في المشروع:

```sql
-- استبدل USER-ID و PROJECT-ID بالقيم الصحيحة
INSERT INTO user_projects (user_id, project_id, project_role)
VALUES ('USER-ID', 'PROJECT-ID', 'owner');
```

### إضافة فريق العمل:

للسماح للمهندسين والمقاولين بالدخول، كرر الخطوات:
1. أنشئ مستخدم في Supabase Auth
2. أضف سجل في `user_profiles` مع الدور المناسب
3. أضف سجل في `user_projects` لربطه بالمشروع

## ❓ استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| **الصفحة بيضاء** | تأكد أن ملف `.env` صحيح وأن `npm install` تم بنجاح |
| **خطأ في تسجيل الدخول** | تأكد من إنشاء المستخدم في Supabase Auth وليس فقط في `user_profiles` |
| **خطأ 404 في التصفح** | تأكد من وجود ملف `_redirects` في مجلد `public` |
| **الترجمة لا تعمل** | اذهب لـ `Settings` → غيّر اللغة إلى العربية أو الإنجليزية |
| **القائمة الجانبية فارغة** | تأكد من تشغيل seed data في الخطوة 2 |
| **الصفحات لا تظهر** | قد تحتاج لتفعيل الوحدات من System Designer |
| **نسيت كلمة سر Supabase** | اذهب لـ Supabase Dashboard → Project Settings → Database → Reset password |
