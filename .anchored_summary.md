# Session: Fix 5 Persistent UI/Data Issues (Round 3)

## Root Cause Discovered
**The Supabase units query was failing with 400** because it selected `sale_price` which doesn't exist in the `units` table (it's in `unit_sales`). This blocked ALL unit data from loading ‚Üí no labels, no markers, no table entries.

## Changes This Round (commit `7983b51`, deployed)

### Fix 1: Units query 400 error (CRITICAL)
- **MapsPage.tsx:1796** ‚Äî removed `sale_price` from `supabase.from('units').select(...)`
- **ProjectSitePlan.tsx:109** ‚Äî same fix
- **MapUnit interface** ‚Äî removed `sale_price` field
- **Popup references** ‚Äî changed `u.sale_price || u.price` to just `u.price`

### Fix 2: Labels now show trailing number (e.g., "001")
- All marker shortLabels: `u.unit_code.match(/\d+$/)?.[0] || u.unit_code.slice(-3)` ‚Äî extracts trailing digits
- All label overlays: same regex ‚Äî just the number part

### Fix 3: 3D view improvements
- `PCFSoftShadowMap` ‚Üí `PCFShadowMap` (deprecation fix)
- Box sizes: max 16‚Üí24, min 2‚Üí4 (sites), 1.5‚Üí3 (buildings)

- Slab height: 0.15‚Üí0.2, opacity 0.5‚Üí0.6
- Building material metalness: 0.3‚Üí0.35
- **Memory leak fixed**: added `cleanupRef` to properly dispose Three.js scenes on re-render/unmount (was leaking renderers/animation loops)
- Sprite labels: larger scale (4√ó1), higher position (0.8z)

### Fix 4: Leaflet `_leaflet_pos` crash
